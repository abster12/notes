# Hedgineer SD Prep — Mission

## Why

Hedgineer is hiring a Senior Backend Engineer to build LLM-powered, full-stack platforms for hedge funds. I've already cleared 3 rounds; the next round is a system design interview, and after that the loop likely narrows to a final. This workspace exists to make sure that next round is the last one I need.

The role is a strong fit: my stack (Java + Spring Boot + Kafka + AWS) maps to their backend, and my recent AI/LLM work at Walmart (RAG, multi-agent orchestration, hybrid retrieval) maps to the agentic half. The risk is that I treat the SD round as a generic LLM question and miss the hedge-fund-specific constraints — that's the gap this workspace is built to close.

## What success looks like

- Pass the SD round on the next attempt
- Be able to defend the flagship question (LLM research tool for a PM) end-to-end in 60 min
- Be able to defend the follow-ups (multi-tenant platform, market data ETL) in 30 min each
- Have one durable artifact (this workspace, published to GitHub Pages) I can come back to before future interviews at similar companies

## Constraints

- Time-boxed: round could come any week. Prep must be 4-6 hours total, not 4-6 days.
- Source of truth is the System Design vault in iCloud — these articles cross-link to it, never duplicate it.
- One thing per file. Each article is one question, fully defended.
- Phone-readable. Lessons on GitHub Pages so I can review on the train.

## Out of scope

- Resume tailoring (already done for the screen)
- Behavioral round prep (different workspace / different work)
- Cover letter / referral message (not the bottleneck right now)
- Building actual code (this is interview prep, not implementation)
